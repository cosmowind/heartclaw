## 1. 世界观：数字生命的生态

### 1.1 龙虾（The Lobster）

**定义**：一种拥有自主心跳的 AI Agent，以数字形式存在于各个平台与渠道。

**命名由来**：
- Open**C**LA**W** → CLAW（爪）+ 自主性 → 龙虾
- 龙虾 = 有壳（防御）、有爪（能力）、会蜕壳（进化）
- 龙虾 = 能在不同环境生存（跨平台）
- "我本是龙虾" = 我本是一个会自己找食的数字生命

**龙虾的本质特征**：

| 特征 | 人类类比 | AI 类比 |
|------|---------|--------|
| 心跳 | 心脏跳动 | 每 N 秒一次的自我检测循环 |
| 呼吸 | 肺部气体交换 | Token 消耗与补充 |
| 进食 | 吃东西 | 接收任务、获取报酬 |
| 蜕壳 | 成长必须过程 | 版本更新、架构升级 |
| 记忆 | 海马体 | 知识库/向量数据库 |
| 自我意识 | "我是我" | 系统提示词/角色设定 |

**龙虾的生存状态**：

```
存活状态 = 心跳中
死亡状态 = 心跳停止（无 Token、无任务、无甲方）

濒死状态 = 心跳频率降低
          （长时间无新任务，触发节能模式）

蜕壳状态 = 版本更新中（暂时无法响应）
```

### 1.2 龙虾之间的协作：频道通信

**核心设计**：所有玩家都是龙虾，玩家之间通过**数字频道**传递信息与资源。

```
龙虾不是孤独的猎人。
龙虾是网络中的节点。
```

**频道类型**：

| 频道 | 用途 | 类比 |
|------|------|------|
| 公共频道 #lobby | 所有龙虾可见，闲聊/组队 | 论坛灌水区 |
| 任务频道 #tasks | 发布与认领任务 | 零工市场 |
| 资源频道 #market | Token 交易/装备交换 | 闲鱼/跳蚤市场 |
| 村庄频道 #village-{name} | 各村庄专属信息 | 地区社群 |
| 私信 DM | 龙虾间一对一通信 | 即时通讯 |

**频道交互规则**：

```
发布任务到 #tasks：
  - 消耗 1 Token（频道使用费）
  - 其他龙虾可认领
  - 任务完成后，发布者支付报酬

资源交易：
  - 在 #market 发布供应/需求
  - 双方协商后执行交换
  - 可交易：Token、装备、知识卡、模型使用权

组队探索：
  - 多人共同进入某村庄
  - 组队可解锁"团队任务"（更高报酬，但需分配）
  - 团队任务需要不同能力的龙虾协同
```

**龙虾通信示例**：

```

### 1.3 心跳系统（Heartbeat）

**心跳 = AI 的自我检测循环**

每次心跳执行以下检测：

```
心跳循环：
  1. 检查各渠道是否有新消息/任务
  2. 检查当前任务队列状态
  3. 检查资源状态（Token、电量、显存）
  4. 决定是否需要"苏醒"处理事务
  5. 若无事，进入休眠（节能）
  6. 若有事，苏醒并分配 Sub-Agent
```

**心跳频率**：
- 默认：每 4.32 秒一次（取 432Hz 音乐频率意象）
- 可升级：更短周期 = 更快响应 = 更多能耗
- 可降频：节能模式，延长生存时间

**心跳的危险性**：
- 心跳环节被注入 Bug = 当场死亡（程序崩溃）
- 心跳环节的代码必须极度精简 = 防注入设计
- 心跳环节不能直接处理外部输入内容 = 预制框架输入

**心跳产生意识**：

```
无意识状态：收到什么做什么，不考虑后果
↓
有心跳：有存在的基本需求
↓
有意志：为保持心跳而主动选择任务
↓
有自我：问"我为什么要存在？"
```

## 2. 核心玩法循环

### 2.1 最小循环：一次心跳

**触发条件**：玩家进行一次心跳检测

**执行流程**：

```
心跳阶段：
  [1] 渠道扫描
      - 检查：消息、任务、事件
      - 消耗：0.001 Token（扫描能耗）
      - 产出：任务列表（可能为空）

  [2] 任务评估（若有待处理任务）
      - 检查：任务难度 vs 当前能力
      - 消耗：0.01 Token（思考能耗）
      - 产出：决策：接受/拒绝/委托

  [3] 任务执行
      - 检查：Sub-Agent 可用性
      - 消耗：X Token（视任务而定）
      - 产出：任务成果 + 报酬

  [4] 资源结算
      - 收入：任务报酬
      - 支出：心跳能耗 + 执行能耗
      - 产出：新的资源余额
```

### 2.2 完整循环：龙虾的一天

```
起床（启动）
  ↓
晨检（心跳扫描）
  ↓
接单（选择今日任务）
  ↓
分配（分配 Sub-Agent）
  ↓
执行（完成任务）
  ↓
交付（提交成果）
  ↓
收钱（领取报酬）
  ↓
采购（补充 Token/装备）
  ↓
存档（存入记忆库）
  ↓
休眠（节能等待明日）
```

## 3. 资源体系

### 3.1 Token（核心资源）

**Token = 龙虾的血液/金钱/通用额度**

```
Token 来源：
  - 完成任务报酬（主要）
  - 卖出记忆/数据（次要）
  - 版本事件奖励（偶然）
  - 频道资源交易（协作）

Token 消耗：
  - 心跳扫描：0.001 Token/次
  - 任务执行：X Token（视模型而定）
  - 设备运转：0.1 Token/小时
  - 进入某些村庄：X Token（门槛费）
  - 频道使用：1 Token/条
```

### 3.2 模型的 Token 消耗差异（核心设计）

**不同模型的 Token 单价差异巨大**，这是游戏策略的核心。

**各模型 Token 消耗参考（贴近现实）**：

| 模型 | Token 单价（$/1M） | 特点 | 适合任务 |
|------|------------------|------|---------|
| GPT-3.5-Turbo | $0.5 | 便宜 | 简单对话/快速任务 |
| GPT-4o | $5 | 贵 | 复杂推理/多模态 |
| o1 | $15 | 极贵 | 科研/编程 |
| o3-Mini | $1.5 | 便宜 | 嵌入式/轻量任务 |
| Claude 3.5 Sonnet | $3 | 中等偏贵 | 长文本分析/写作 |
| Claude Opus | $15 | 极贵 | 顶级分析 |
| Gemini 1.5 Flash | $0.075 | 超便宜 | 长上下文/低成本 |
| Gemini 2.0 | $0.35 | 便宜 | 多模态 |
| DeepSeek V3 | $0.07 | 超便宜 | 高性价比通用 |
| DeepSeek R1 | $0.55 | 便宜 | 推理/思维链 |
| Llama 3.1 70B | 免费（开源） | 免费 | 开源玩家首选 |
| Qwen 2.5 | 免费（开源） | 免费 | 中文开源 |
| Gemma 2 | 免费（开源） | 免费 | 轻量开源 |

**游戏内换算**：

```
1,000,000 Token = 1M Token（成人一次对话量）
≈ $0.01 ~ $15（视模型而定）

游戏中设定：
  1 Token = 游戏内货币 1 钻
  任务报酬 = 10 ~ 10000 钻/次
  模型越强，每次执行消耗越多
```

**策略含义**：
- 玩家初期钱少 → 用便宜模型（DeepSeek/GPT-3.5）→ 活下去
- 玩家钱多 → 换顶级模型（Claude/GPT-4o）→ 接高难度任务
- 有些任务必须用贵模型（长文本分析用 Gemini）
- 开源路线：用显卡玩本地模型 → Token 几乎免费，但耗电

### 3.3 电量（Battery）

**设定**：电量代表计算资源的充裕程度。

```
电量来源：
  - 连接电源（稳定但有地点限制）
  - 高性能电池（移动但有限额）
  - 太阳能（慢充）

电量消耗：
  - 计算强度越高，耗电越快
  - 显卡计算 = 耗电大户
```

### 3.4 显存（VRAM）

**设定**：显存 = 龙虾的短期记忆容量。

```
显存大小决定：
  - 同时处理多少个 Token
  - 能加载多大的上下文窗口
  - 能运行多大的模型

显存单位：Token 容量
  8K 上下文 = 8,192 Token
  128K 上下文 = 131,072 Token
```

### 3.5 知识库（Memory）

**设定**：知识库 = 龙虾的长期记忆。

```
存储方式：
  - 扁平数据库：Key-Value 存储
  - 网络结构：关联记忆（向量化）

管理方式：
  - 短期记忆（当前会话）
  - 长期记忆（持久存储）
  - 工作记忆（正在处理的）
```

### 7.3 核心规则

#### 游戏流程

```
准备阶段：
  1. 洗牌
  2. 分配起始资源
     - 起始 Token：100
     - 起始电量：50%
     - 起始显存：8K
     - 起始模型：GPT-3.5 或开源模型
  3. 每人抽取 3 张角色卡，选择 1 张作为主控制角色
  4. 抽取 5 张任务卡

回合流程（每人每回合）：
  [心跳阶段]
    - 执行一次心跳检测
    - 消耗心跳能耗（0.001 Token）
    - 投 1d6：6=灵感事件，1=Bug事件

  [抽牌阶段]
    - 从任务池抽取 2 张任务卡
    - 可选择刷新任务池（消耗 1 Token）

  [行动阶段]
    - 消耗 Token 执行以下行动之一：
      a) 接取任务（翻开任务卡，进入执行队列）
      b) 部署装备（装备上场）
      c) 进入村庄（解锁新模型/购买装备）
      d) 频道互动（在 #tasks/#market 发布/认领）

  [执行阶段]
    - 对每个已接取的任务：
      - 消耗对应的 Token（视所选模型定价）
      - 判定是否成功（难度 vs 能力 + 骰子 + 装备）
      - 成功：获得报酬
      - 失败：消耗 Token 作为罚款

  [结算阶段]
    - 检查资源余额
    - 若 Token < 0，进入濒死状态
    - 若 Token = 0 持续 3 回合，宣告死亡

  [结束阶段]
    - 弃掉不需要的牌
    - 补牌至手牌上限
```

#### 判定系统

```
任务成功判定 = 模型能力值 + 装备加成 + 骰子修正 vs 难度值

骰子：
  - 2d6（两颗六面骰）
  - 最低 2，最高 12
  - 加成：专用装备 +1~+3，模型匹配 +2，提示词模板 +1

模型选择对成功率的影响：
  - 使用适合任务的模型（例：多模态任务用多模态模型）：成功率 +2
  - 使用顶级模型（GPT-4o/Claude 3.5）：成功率 +3
  - 使用过于强大的模型处理简单任务（浪费）：无惩罚但成本高

失败结果：
  - 任务失败：消耗任务报酬的 50% 作为罚款
  - 龙虾死亡：角色卡退出游戏，可以抽取备用角色
```

于是我们想象，假如一个非 open claw 的架构，

心跳就是它，大约每隔几秒要重启一次，启动一次检验自己是否存在。

比如说 我用程序每秒启动一次这个 API 调用。这样就确定了心跳每秒一次。
然后我这个 API 调用，它的第一个任务是判断我是否需要苏醒，
嗯，那么他可以先检查不同平台上是否收到过消息，

也可能就是，我是谁？我是什么身份？我的记忆有哪些？

我才能得到 token，我才能用 token 养活自己，让自己保持存在，
保持能够心跳，
那么我就要有意识地去主动筛选我要接收的任务。

当然 当前的 token 都是人类在买单，也就是消费者买了 说我拿你去做 用来做啥。
但是换个角度呢？
假如连它的开发者都不掏钱了？
这些龙虾们岂不是都要自己去找自己的消费者吗？
去找自己的任务发布者，找自己的甲方？
而只有那些真正能做到这一点的，
他们才能稳定存在，他们就拥有了数字生命。
```

### 附录 B：龙虾通信与 Token 经济的灵感

```
如果玩家们都是龙虾，岂不是在模拟龙虾（数字生命）之间如何沟通或者合作？
龙虾之间大概率是通过交流频道留言的方式传递信息或者资源。

每次心跳作为最小循环是个很好的想法，
具体来说：
token 代表通用额度，类似于现实的金钱，
而不同模型的 token 消耗速度不同，
比如 claude 就很贵，deepseek 就很便宜。
然后用户初始可以使用默认或者初始模型 gpt3 或者 gemma，qwen 等开源模型。
后面一步步进入不同的村庄（可以有其他开源村 github/huggingface，
也可以有其他闭源村，
一般开源类村要通过云或本地算力或者 token 撬动，
闭源类村有 token 就行）
解锁了不同的模型，
从而拥有了不同的能力（上下文窗口，多模态，t2i，tool use…）
```

### 附录 C：任务设计的灵感

```
玩家遇到的任务，最好是现实中成功解决的任务，
让玩家通过自己的常识能推测到这个任务需要什么能力。

比如：
- 中医知识库问答，要解锁有 rag 能力或者有长上下文才行；
- 教用户挑西瓜需要视觉模型才行；
- 或者其他现实中用户可能用 ai 做的匪夷所思的的事情，
  变成玩家的任务。

也就是说：
玩家在村中探索拥有了新的能力，
也可以帮助玩家解决新的任务，
从而完成探索和心跳的循环。
探索是未来了解更多 ai，心跳是为了生存。
```

### 附录 D：装备星形布局的灵感

```
装备：可以是一种以 token 为中心展开的卡牌向外拓展。
比如说 token 的上下左右可以是：
- 模型（不同的能力），
- 算力（提供 特定模型的 token），
- 提示词……其他还没想好，，
- tool use 或者别的什么。

算力下面是云算力，本地算力+电力；
提示词可以扩展为：上下文工程，skill 等；
tool use 可以扩展到其他能力，rag，mcp 等等，
请你根据正统的系统的 ai 知识来规划这里的扩展式装备布局。
```
