## 四、桌游规则书优先设计

### 4.1 最小可玩游戏（两人/30分钟）

```
目标：验证核心循环（心跳→接任务→消耗Token→完成→结算）

所需卡牌：
  - 角色卡 × 6（每人有3选1）
  - 任务卡 × 12
  - 装备卡 × 6
  - 村庄卡 × 3

简化规则：
  - 无频道系统（单人或两人对战）
  - 无电量/显存（简化资源）
  - 无版本事件（第一版不含）
  - 每回合固定：心跳→抽2任务→执行1任务→结算
```

### 4.2 完整规则书目录

```
1. 游戏概述
2. 准备与设置
3. 回合结构（心跳→抽牌→行动→执行→结算→结束）
4. 心跳详解（0.001 Token + 骰子判定）
5. 任务系统（接取→判定→完成/失败）
6. 装备系统（部署→生效→组合特效）
7. 村庄系统（进入→解锁模型/购买装备→离开）
8. 频道系统（任务发布→认领→交易）
9. 资源系统（Token/电量/显存）
10. 胜利与失败
11. 高级规则（版本事件/声望/团队任务）
12. 索引与术语表
```

## 六、知识渗透教案（知识卡如何使用）

### 6.1 学习路径设计

```
第一阶段：新手村（完成前5个任务后）
→ 自动解锁：Token/上下文窗口/提示词/生成式AI/API

第二阶段：成长期（进入第二个村庄后）
→ 自动解锁：Function Calling/多模态/RAG/思维链

第三阶段：扩张期（完成高难度任务后）
→ 自动解锁：Agent/MCP/微调/Embedding

第四阶段：高级期（声望达到3星后）
→ 自动解锁：所有底层知识卡（Transformer/Scaling Law等）
```

### 6.2 知识卡使用方式

```
方式1：任务嵌入
完成任务 → 解锁对应知识卡

方式2：村庄购买
进入对应村庄 → 可购买相关知识卡

方式3：成就解锁
特定成就 → 解锁稀有知识卡

方式4：交易
在#market频道 → 用Token购买他人的知识卡
```
